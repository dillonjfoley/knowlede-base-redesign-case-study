# Metadata and Tagging Model

Metadata helps the knowledge base stay organized, searchable, and maintainable.

## Required Metadata Fields

| Field | Purpose | Example |
|---|---|---|
| Article type | Identifies the content format | How-to |
| Audience | Shows who the article is for | Employee |
| Owner | Names the team responsible for the article | IT Service Desk |
| Review cycle | Defines how often the article is reviewed | Quarterly |
| Last reviewed | Shows the most recent review date | 2026-06-20 |
| Tags | Improves search and filtering | vpn, mfa, remote-access |

## Article Types

| Type | Use For |
|---|---|
| How-to | Step-by-step task instructions |
| Troubleshooting | Fixing a known issue |
| Reference | Policy, definitions, or background information |
| FAQ | Common questions and short answers |

## Audience Values

| Audience | Description |
|---|---|
| Employee | General internal users |
| Service desk agent | Tier 1 or Tier 2 support |
| Manager | People managers or approvers |
| System administrator | Technical system owners |

## Recommended Tags

| Tag | Use For |
|---|---|
| password-reset | Password reset articles |
| mfa | Multi-factor authentication articles |
| vpn | VPN setup or troubleshooting |
| remote-access | Remote work access issues |
| software-access | Software request articles |
| shared-folder | Shared drive or folder access |
| laptop | Laptop setup or troubleshooting |
| email | Email and calendar support |

## Governance Rules

- Every article must have an owner.
- Every article must have a review cycle.
- Tags should match user search terms.
- Do not create duplicate tags with the same meaning.
- Review outdated articles before creating new ones.

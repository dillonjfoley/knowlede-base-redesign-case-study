# Before Taxonomy

The original knowledge base was organized by product, platform, or internal system name.

| Top-Level Category | Subcategory | Problem |
|---|---|---|
| VPN Client | Install VPN | Users may not know the VPN product name. |
| VPN Client | VPN Errors | Troubleshooting articles were mixed with setup articles. |
| Identity Manager | Password Reset | Users searched for "forgot password" instead. |
| Identity Manager | MFA | MFA articles were separated from login help. |
| Ticketing Portal | Submit Ticket | Users searched for "request help" instead. |
| Device Management | Laptop Setup | Setup articles were mixed with repair articles. |
| Email System | Outlook Issues | Articles were grouped by tool, not task. |
| File Storage | Shared Drive Access | Users searched for "request folder access." |

## Main Issues

- Product names were used as categories.
- User tasks were buried under system names.
- Similar content existed in several places.
- Article titles did not follow one pattern.
- Metadata was missing or inconsistent.

# After Taxonomy

The redesigned knowledge base is organized by task and user need.

| Top-Level Category | Subcategory | Example Article |
|---|---|---|
| Access and Passwords | Password Help | Reset your network password |
| Access and Passwords | Multi-Factor Authentication | Set up multi-factor authentication |
| Network and VPN | VPN Setup | Connect to VPN for the first time |
| Network and VPN | VPN Troubleshooting | Fix VPN connection errors |
| Software Requests | Request Software | Request access to approved software |
| Hardware and Devices | Laptop Support | Prepare your laptop for remote work |
| Email and Calendar | Email Troubleshooting | Fix Outlook sync issues |
| File Access and Storage | Shared Folder Access | Request access to a shared folder |

## Why This Works Better

- Users can browse by task.
- Search terms match common user language.
- Troubleshooting content is easier to separate from how-to content.
- Content owners can manage smaller article groups.
- Duplicate content is easier to find and remove.

# Article Title

## Summary

Briefly explain what this article helps the user do.

## Audience

Who should use this article?

Example:

- Employees
- Service desk agents
- Managers
- System administrators

## Before You Start

List anything the user needs before they begin.

Example:

- Active network account
- Company laptop
- Internet connection
- Manager approval

## Steps

1. Step one.
2. Step two.
3. Step three.

## Expected Result

Explain what should happen when the user completes the steps.

## Troubleshooting

| Issue | Cause | Fix |
|---|---|---|
| Example issue | Example cause | Example fix |

## Escalation

Contact the service desk if:

- The issue continues after following the steps.
- You receive an error message not listed in this article.
- You do not have the required permissions.

| Review cycle | Quarterly |

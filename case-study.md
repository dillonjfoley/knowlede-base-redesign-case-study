# Case Study: Reorganizing an Internal IT Knowledge Base

## Summary

This fictional project shows how an internal IT knowledge base was redesigned from a product-based structure into a task-based structure.

The original knowledge base was hard to search because users needed to know the name of the system before they could find the right article. The redesigned structure focuses on what users are trying to do, such as reset access, troubleshoot a connection, request software, or fix device issues.

## Problem

The old knowledge base had several issues:

- Articles were grouped by product name instead of user task.
- Duplicate articles existed across multiple product folders.
- Article titles used inconsistent wording.
- Some articles did not have owners or review dates.
- Troubleshooting steps were written in different formats.
- Search results returned too many similar articles.

## Redesign Approach

The redesign used four main steps:

1. Audit existing articles.
2. Group articles by user task.
3. Create standard article templates.
4. Add metadata for ownership, audience, topic, and review cycle.

## Before

The old taxonomy used product-based categories, such as:

- VPN Client
- Identity Manager
- Ticketing Portal
- Device Management
- Email System
- File Storage

This structure worked for IT teams but not for employees who did not know which system handled their issue.

## After

The new taxonomy uses task-based categories, such as:

- Access and Passwords
- Network and VPN
- Software Requests
- Hardware and Devices
- Email and Calendar
- File Access and Storage

This structure helps users find content based on what they need to do.

## Results This Redesign Was Intended to Support

Because this is a fictional portfolio project, the results below are sample target outcomes rather than real company metrics.

- Fewer duplicate articles
- Faster article discovery
- More consistent troubleshooting steps
- Clearer ownership for each article
- Easier review and maintenance
- Better alignment between article titles, tags, and user search terms

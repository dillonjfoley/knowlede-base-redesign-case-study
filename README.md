# Knowledge Base Redesign Case Study

This fictional case study shows how an internal IT knowledge base was reorganized from product-based categories into task-based categories.

The goal was to make support content easier to find, easier to maintain, and easier to govern.

## Scenario

Northstar Credit Services had an internal IT knowledge base used by support agents and employees. The original structure was organized by product names and internal system names. This made sense to system owners, but it was hard for users to find answers when they did not know which product caused their issue.

This redesign changes the structure from product-based categories to task-based categories.

## Project Goals

- Improve findability
- Reduce duplicate articles
- Standardize article names
- Create reusable article templates
- Add metadata for ownership and maintenance
- Make troubleshooting and how-to content easier to scan
